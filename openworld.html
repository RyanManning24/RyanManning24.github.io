Breif Overview of Development

I created an infinitely generating world with NPC’s, which randomly move around the world, and a randomly spawning house. Here you can see a video of this working.

The infinitely generating world is created using chunks. To start this, I first implemented a file reading system that would be able to read text files and use that information to be able to generate objects in the game world.

Once this was achieved my job was to chunk up the world so that I can make sure that only the area around the player is loaded in. The main problem with this was that occurred was unloading chunks and making sure that as the player moved away the floor would unload. As this left unchecked would just cause the game to continually get harder to run as the player would load more chunks in. To fix this issue and try to optimise for the future I decided to implement a system where instead of unloading the floor and loading the floor back in each time to just move unused floor around the player. This then saves on loading costs in the game and creates the illusion that the game is unloading everything like normal.

This report can be found and download here.

The NPCs in this game uses a waypoint system that allows the NPC’s to randomly move around the loaded in area. One problem that could occur with this is that if the NPC runs into a wall trying to get to their waypoint, they will be infinitely stuck. To overcome this issue, I created a fail safe so that if the NPC registers that it’s been in the same spot for a couple of seconds then the waypoint that they were moving to is scrapped and a new one is generated.
